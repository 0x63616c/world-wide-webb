Control Center

Control Center becomes one product instead of the repository identity. It keeps the wall panel, home integrations, API, workers, media worker, Storybook, Drizzle tooling, and iOS hosted kiosk shell, but its deployment and data boundaries are product-owned.

  • Runs on homelab k3s.
  • Depends on Home Assistant, UniFi, NAS media, and home network integrations.
  • Uses private Cloudflare Access exposure for app and any external API surface.
  • Uses its own CNPG Postgres cluster and platform-managed backups.
  • Must continue working throughout the migration.

Captive Portal

Captive Portal becomes a full product with its own frontend, API, backend storage, UniFi integration boundary, database, backups, and deployment. It is no longer a frontend plus routes inside Control Center API.

  • Runs on homelab k3s because it depends on UniFi guest network behavior and LAN reachability.
  • Uses captive LAN exposure, not generic public web exposure.
  • Owns portal guest/code/attempt/authorization data in its own product database.
  • Redesigns its backend now rather than preserving the accidental Control Center coupling.
  • Must preserve real guest onboarding, Apple captive network assistant behavior, TLS, split DNS, firewall/walled-garden rules, and UniFi authorization.
Product boundaries are platform contracts. Avoid hidden coupling across product folders.
