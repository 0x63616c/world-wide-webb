M1 Shipped Foundation

@repo/platform is the representation layer, not the live deploy path yet. It is a pure TypeScript package that proves Control Center can be expressed through platform primitives without changing production behavior.

  • defineProduct() derives folder, namespace, DNS code, labels, Pulumi names, image repositories, service names, and backup path parts from one product slug.
  • homelabTarget is the only implemented target. Future targets are explicit unsupported states, not silent fallthroughs.
  • publicWeb(), privateWeb(), captivePortalWeb(), and internalService() model exposure intent only. M1 does not mutate Cloudflare, cert-manager, UniFi, or LAN forwarding.
  • defineServiceSecretUsage() and secretCatalog model explicit service secret usage from 1Password Homelab refs, with no secret values in code.
  • defineProductDatabase() models per-product CNPG intent and can render the current Control Center database shape in compatibility mode.
  • defineDatabaseBackup() makes database backups mandatory and can render the current pg-backup behavior while also deriving the future platform NAS path.
  • controlCenterProductManifest() is the M1 capstone fixture proving one existing product can be represented by the new platform model.

App Definition

The app context should be the source of truth. Avoid passing owner, product, namespace, backupPath, or magic string identifiers into every primitive.

const textYourEx = app({
  slug: "text-your-ex",
});

textYourEx.service("app", {
  image: images.product("app"),
  exposure: publicWeb({ host: "app" }),
});

textYourEx.service("api", {
  image: images.product("api"),
  exposure: publicWeb({ host: "api" }),
});

textYourEx.database({
  size: "5Gi",
});

The app definition derives products/text-your-ex, namespace text-your-ex, image names, labels, secrets, Pulumi names, and backup paths. Networking primitives derive DNS labels from product names instead of passing short labels through general app code, so publicWeb({ host: "api" }) can produce api.tye.worldwidewebb.co and the required TLS coverage.

Customization Rule

Defaults are platform-owned. Only expose customization when the app team has a real product reason.

If every app should do it, it is not configurable. Backups, naming, labels, secret mount wiring, and route mechanics should not be exposed as app knobs.

Size is a valid knob. Product data volume differs, so database storage size should be configurable.

Core Primitives

app()

Declares product identity once. All derived names come from this context.

service()

Declares a deployable workload or static web service. Product code should not hand-write Kubernetes Deployment and Service objects.

database()

Creates a per-product CNPG Postgres cluster, credentials, service discovery, backup job, and restore contract.

exposure

Declares public web, private web, captive portal, or internal service intent. Platform implements Cloudflare, Access, cert-manager, UniFi, service routing, and exact-host or product-wildcard TLS.

secret()

Declares which service can access which central 1Password-backed secret. Shared secrets are allowed when intentionally shared.

target()

Declares runtime target. Homelab k3s is the only implemented target in this migration, but the platform should not be homelab-blind.

Do Not Expose

  • Backup enabled or disabled.
  • Backup path.
  • Backup retention.
  • Namespace naming pattern.
  • Label schema.
  • External Secrets low-level wiring.
  • Default secret names.
  • Cloudflare record comments and implementation details.
  • Low-level Kubernetes object shapes for normal apps.

Expose Carefully

  • database.size, because data volume differs by product.
  • Service port, because runtime servers listen on different ports.
  • Resource presets, preferably small, medium, large before raw CPU/memory everywhere.
  • Exposure class, because public/private/captive/internal is product intent.
  • Declared secret usage, because least-privilege requires explicit access.
The platform API should be easy to read in product code. If every product repeats the same low-level shape, extract it.
