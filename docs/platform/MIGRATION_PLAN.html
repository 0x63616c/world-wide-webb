Milestones

Break the migration into production-safe lanes.

The migration should create stable platform seams first, then let product lanes run in parallel. Most tasks should deliver something verifiable in production. Internal refactor tasks are allowed when they create a safe seam.

Work Order Principle

Default every task to a production deliverable with tests, verification, and rollback notes. Allow internal foundation tasks only when a safe seam must exist before anything can ship.

Parallelization is a goal, but only after shared contracts are stable. Do not let multiple lanes casually edit the same product, infra, or CI files.

Milestone 0: Persist The Plan

  • Create the platform docs in docs/platform/.
  • Add a concise platform principle to agent instructions.
  • Create Beads epics and child tickets from this plan.
  • Acceptance: docs are checked in, linked, and future agents know the platform-team rules.

Milestone 1: Platform Foundation

  • Create packages/platform or equivalent shared platform package for product definitions and deploy primitives.
  • Define product identity once: slug, DNS code, namespace, image namespace, labels, and Pulumi names.
  • Define exposure classes: public web, private web, captive portal, internal service.
  • Define target model with homelab as the only implemented target.
  • Define database primitive as per-product CNPG Postgres cluster with automatic backup.
  • Define secret catalog and explicit service secret usage.
  • Acceptance: one existing product can be represented in the new model without behavior change.

Milestone 2: Data Safety Baseline

  • Generalize the current pg-backup pattern into the platform backup primitive.
  • Keep current Control Center backups running during the migration.
  • Take a pre-migration custom-format Postgres dump before any data movement.
  • Restore that dump into a scratch database and validate row counts.
  • Document restore commands and verification checks.
  • Acceptance: current production data has a proven restore path before product extraction touches data ownership.

Milestone 3: Networking And TLS Contract

  • Model all route exposure through platform primitives.
  • Preserve TLS/cert behavior during hard hostname cutover.
  • Make Cloudflare Access the private-web implementation.
  • Make Captive Portal a special UniFi/captive exposure, not generic LAN web.
  • Move old hostnames to the new app.<product> surfaces and only create external API hostnames where a product explicitly declares one.
  • Acceptance: dashboard, captive portal, Storybook/Drizzle-style private routes, and TLS all verify on the new model.

Milestone 4: Product Folders And CI Split

  • Create products/control-center, products/captive-portal, products/text-your-ex, and products/amp.
  • Move product-owned code behind those boundaries.
  • Introduce separate product workflows for CI/deploy.
  • Optimize enough for product isolation, not perfect speed.
  • Acceptance: changing Text Your Ex does not rebuild/deploy Control Center unless shared platform code changed.

Milestone 5: Captive Portal Product

  • Split Captive Portal out of Control Center.
  • Redesign its backend/API/storage as product-owned.
  • Move portal data from Control Center Postgres into Captive Portal Postgres with export, rehearsal, final snapshot, validation, and rollback.
  • Preserve UniFi guest onboarding, split DNS, cert-manager TLS, firewall/walled-garden rules, and Apple CNA behavior.
  • Acceptance: real guest onboarding works on the production guest network through app.cp.worldwidewebb.co.

Milestone 6: Text Your Ex Product

  • Move current text-your-ex app into products/text-your-ex.
  • Ship app and API to production on homelab k3s.
  • Move from current SQLite prototype to product Postgres.
  • Ensure production starts empty, while local dev can still seed demo data explicitly.
  • Standardize bundled iOS TestFlight workflow with VITE_API_BASE pointing at api.tye.worldwidewebb.co.
  • Acceptance: web production route works, API health works, database persists, backups run, and TestFlight build targets the production API.

Milestone 7: Control Center Product

  • Make Control Center product-owned without breaking the wall panel.
  • Keep home integrations working through the product boundary.
  • Cut Control Center over to its product CNPG cluster with export, restore rehearsal, row-count validation, final snapshot, and rollback.
  • Update the hosted iOS kiosk URL to the new private route.
  • Acceptance: wall panel loads at the true 1366x1024 target, no skeleton/error regressions, workers run, and the iOS kiosk build points at the new URL.

Milestone 8: AMP Product

  • Build and deploy minimal AMP as a real product.
  • Use private Cloudflare Access exposure.
  • Prove the platform can stand up a new app with the standard product path.
  • Acceptance: AMP has app route, optional API route, CI/deploy, logs, health, and platform-owned database/backups if it declares a database.

Milestone 9: Rename And Polish

  • Rename repository to world-wide-webb after production state is stable.
  • Update docs, badges, CI labels, image naming, Pulumi project naming, and references.
  • Acceptance: all products remain deployed, CI remains green, and no old repo identity remains in load-bearing paths.
