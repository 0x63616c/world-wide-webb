Developer experience

Every product should be easy to run locally.

Local Tilt and iOS ergonomics are part of the platform. Product developers should not rediscover how to start services, open local URLs, sync Capacitor, or run an iOS simulator.

Tilt Standard

  • Each product gets its own Tilt entry.
  • An umbrella Tiltfile can run one product without starting every product.
  • Product Tilt resources should have clear labels: setup, backend, frontend, database, tooling.
  • Readiness probes should exist where services expose health endpoints.
  • Local dev should use product-scoped databases and ports.
  • Local secrets should use the established 1Password loading pattern, not committed env files.

Standard iOS Commands

Future iOS products should use the same command names. Tilt should expose manual buttons for these actions.

bun run ios:sync
bun run ios:open
bun run ios:sim
  • ios:sync builds/syncs Capacitor assets.
  • ios:open opens the Xcode project.
  • ios:sim runs the app in an available simulator when configured.

Control Center iOS

  • Stays hosted kiosk mode.
  • Updates runtime URL from old dashboard host to the new private app host.
  • Needs TestFlight update as part of hostname cutover.
  • Gets Tilt buttons for sync/open/sim to match the platform standard.
  • Must preserve kiosk behavior, idle timer, landscape, watchdog, and recovery behavior.

Local Product Expectations

Control Center

Runs local Postgres, API, worker, web, Storybook, and iOS shell tools without starting unrelated products.

Captive Portal

Runs portal app/API/database locally. Real guest network verification remains a production or staging network check.

Local dev is part of the platform contract. If it is hard to run locally, the primitive is not done.
