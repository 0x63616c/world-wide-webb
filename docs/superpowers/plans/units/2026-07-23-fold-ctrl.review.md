## Findings

### [MAJOR-1] PLACEHOLDER-1 must be resolved by the manager before implement
The controls service needs `config/lights` + `config/identity`, un-importable from
`features/` (`features ‚Üí apps/api` banned). The plan is correct that they must move to
`@www/core`, and offers **(A)** fold the hoist into the atomic commit vs **(B)** a
prior standalone hoist unit. This is a real decision, not a defect ‚Äî but the implementer
**cannot start until it is picked**. Recommend **A** (fewest commits; registry reaches
its roadmap-final home; device-ownership *logic* stays deferred). Whichever is chosen,
the `packages/core/src/index.ts` edit races other core-touching units ‚Äî serialize the
merge (`git pull --rebase --autostash` immediately before push), as the plan says.
**Fix:** manager records A or B in the plan/dispatch before the implementer is spawned.

### [MINOR-1] `controls.test.ts` imports `router` from `trpc/init` and `controlsRouter` from `routers/controls` ‚Äî repoint not spelled out
`apps/api/src/__tests__/controls.test.ts:66-67` imports `router` from `../trpc/init` and
`controlsRouter` from `../trpc/routers/controls`. The plan's ¬ßTests repoint list
(`./api`, `./service`, `./lamp-scenes`, `./schema`, `@www/core`) omits these two. In the
feature the test must get `router` from `@app-kit/server`, and it needs `controlsRouter`
‚Äî but the plan's `api.ts` snippet keeps `controlsRouter` as a **non-exported** `const`
(only `api` is exported). **Fix:** either export `controlsRouter` from `api.ts`, or
adapt the moved test to build its caller from the exported `api` router. Flag to the
implementer so the sole controls-coverage test compiles.

### [MINOR-2] PLACEHOLDER-3 verified ‚Äî resolves favorably, downgrade to a note
`apps/api/src/trpc/init.ts:49` `export const publicProcedure = t.procedure.use(
haErrorMiddleware)` and `@app-kit/server` re-exports that exact `publicProcedure`. The
feature's `api.ts` using `@app-kit/server`'s `publicProcedure` therefore inherits the
`HaError ‚Üí SERVICE_UNAVAILABLE` mapping automatically ‚Äî no reproduction needed. The
plan's non-blocking gap is closed; implementer need not re-derive.
