# P1.2 — Hoist Sonos Client to `@www/core` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the pure Sonos UPnP/SOAP client out of `apps/api/src/integrations/sonos/` into `packages/core/src/sonos/`, export it from the `@www/core` barrel, and repoint every `apps/api` consumer and test — deleting the old `apps/api` integration folder entirely. This unblocks folding the `sound` + `quickplay` tiles into `features/*` without crossing the Biome-banned `features/* → apps/api/*` boundary.

**Architecture:** This is a mechanical hoist mirroring P1.4 media-path (commit `c4e39bc6b`), NOT the HA P1.1 shim pattern (`da0be339e`). `SonosClient` is a **pure per-IP class** (`new SonosClient(ip)`) with **no env-bound singleton** — the sole reason HA kept an `apps/api` re-export shim. Like media-path, every consumer repoints directly to `@www/core` and the old folder is deleted. The `@www/core` barrel is verified side-effect-free at import time (`db/pool.ts` constructs its pool lazily via `createPool()`, no top-level `new Pool`; `unifi`/`homeassistant`/`sonos` are pure classes/types), so retargeting the test `vi.mock(...)` calls to `@www/core` with `importOriginal` + spread is safe.

**Tech Stack:** TypeScript (ESM), Bun workspaces, Vitest, Biome, knip. No new runtime dependency — the Sonos client uses only global `fetch` / `AbortSignal`, importing nothing beyond its sibling `./errors` and `./types`.

## Global Constraints

- **Design for 10x-100x** (AGENTS.md): data/path layout is the durable decision — `packages/core/src/sonos/` is the permanent home; get it right, code interfaces refactor cheaply.
- **Shim-vs-repoint decision: REPOINT, no shim.** Reason: `SonosClient` has no env-bound singleton (unlike HA's `ha` built from `HA_URL`/`HA_TOKEN`). The one justification for an `apps/api` re-export shim (an env-bound singleton with many consumers) does not apply. - **`@www/core` barrel is the public surface** (`packages/core/src/index.ts`). `apps/api/package.json` already declares `@www/core` (`workspace:*`) — no dependency edit.
- **No new `packages/core` dependency.** The client imports only `./errors`/`./types` and uses global `fetch`. Do NOT touch `packages/core/package.json`.
- **`git mv` for moves** so history follows and no stray copy is left for knip to flag.
