# P1.3 — Hoist Spotify client to @www/core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the env-free Spotify Web API client out of `apps/api/src/integrations/spotify/` into `packages/core/src/spotify/`, export it from the `@www/core` barrel, and repoint every `apps/api` importer — so the quickplay tile (Track C `features/*`) can consume the client without crossing the Biome-banned `features/* -> apps/api/*` boundary.

**Architecture:** Pure code move, mirroring the media-path hoist (P1.4-style, commit c4e39bc6b), NOT the HA hoist shim pattern. The Spotify barrel (`integrations/spotify/index.ts`) is a *pure re-export* — it holds no env-bound singleton (the env binding lives in `spotify-service.ts`'s lazy `getClient()`). Per the shim rule (a shim only earns its keep for an env-bound singleton with many consumers — HA kept shims; media-path did not, and knip zero-tolerance kills unused shims), there is **NO shim**: the old directory is deleted and all importers repoint to `@www/core`. The client unit test moves into `packages/core` alongside the client; the two apps/api service/tRPC tests stay put and switch their `vi.mock` target to `@www/core` via a partial (`importOriginal`-spread) mock so the rest of the core barrel (`createPool`, device-state, secrets) stays real.

**Tech Stack:** TypeScript, Bun workspaces, Vitest, `@www/core` (already declares `@www/logger`, which `client.ts` needs), Biome (boundary rule), knip (dead-code gate).

## Global Constraints

- **NO shim.** Delete `apps/api/src/integrations/spotify/` entirely. Any leftover unused re-export fails knip.
- Barrel is `packages/core/src/index.ts`; `@www/core` package `exports` maps `.` → `./src/index.ts`, so knip treats the barrel as an entry point — re-exported public types are not dead code (the `@public` JSDoc tags carry over as extra insurance).
- `apps/api/package.json` already declares `@www/core` (`workspace:*`). `packages/core/package.json` already declares `@www/logger`. **No dependency edits are required** — `client.ts` imports only `@www/logger` + relative `./errors`/`./types`; `btoa`/`fetch`/`Response` are node/bun globals (core vitest env is `node`).
- No behavior change. No fabricated/placeholder data (the client's throw-on-missing-field semantics are load-bearing — preserve verbatim).
- Structured logging preserved (`getLogger().debug(...)` in `refreshToken`).
- `bun build` reads the **CWD** tsconfig, not the entry's — irrelevant to this unit's local verify (no new Docker `bun build` entrypoint), but do not "fix" import paths to relative deep paths; consumers use the bare `@www/core` specifier.
- Commit + push discipline: `git pull --rebase --autostash` before push (peers push `main` constantly); stage **explicit paths**, never `git add -A`; verify with `git show --stat HEAD` that only your files are in the commit; **no backticks** in `git commit -m`.

## Task 4: Retarget the two service/tRPC tests to a partial `@www/core` mock

**Files:**
- Modify: `apps/api/src/__tests__/spotify-service.test.ts` (line 24 `vi.mock`; 9 dynamic-import sites: 135,142,166,173,193,213,233,261,268)
- Modify: `apps/api/src/__tests__/spotify-browse.test.ts` (line 23 `vi.mock`; 3 dynamic-import sites: 158,165,212)

**Interfaces:**
- Consumes: mocked `SpotifyClient` + real-or-mock `SpotifyError` from `@www/core`.

**Why a partial (spread) mock, not a full replacement:** `vi.mock("@www/core", () => ({ SpotifyClient, SpotifyError }))` would replace the *entire* barrel. `spotify-browse.test.ts` imports the tRPC media router, which transitively imports `apps/api/src/db/index.ts` (`createPool` from `@www/core`) and `apps/api/src/db/schema.ts` (device-state exports from `@www/core`); `spotify-service.ts`'s sibling `apps/api/src/env.ts` imports `databaseUrlFromSecret`/`hydrateSecretFiles` from `@www/core`. A full replace strips all of those and the imports blow up. Spreading `importOriginal()` keeps every real export and overrides only `SpotifyClient` (+ a local `SpotifyError` subclass, matching the current tests).
